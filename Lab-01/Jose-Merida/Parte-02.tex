\section*{Introducción}

En la primera parte de este laboratorio se replicaron esquemas de comunicación previos a las redes modernas (código Morse,
código Baudot y un protocolo propio de conmutación de mensajes), lo cual permitió evidenciar de forma práctica los problemas
de fiabilidad, ergonomía y enrutamiento que enfrentaban dichos esquemas. En esta segunda parte se aborda una herramienta
central para el estudio de las redes de computadoras actuales: Wireshark, un analizador de protocolos que permite capturar
y examinar el tráfico que circula por una interfaz de red en tiempo real.

El objetivo de esta parte es familiarizarse con el entorno de Wireshark, personalizando perfiles, layout y filtros, para luego
realizar una captura de paquetes sobre la interfaz de red activa de la computadora y analizar una solicitud HTTP real, identificando
las versiones del protocolo utilizadas, los encabezados relevantes de la solicitud y de la respuesta, y discutiendo consideraciones
de rendimiento y seguridad relacionadas con el uso de esta herramienta.

\section*{Configuración de la Captura de Paquetes}

\begin{figure}[H]
  \centering
  \includegraphics[width=0.6\textwidth]{figs/ifconfig.png}
  \caption{Ejecución de comando ifconfig.}
  \label{fig:ifconfig}
\end{figure}

El comando \texttt{ifconfig} muestra todos los adaptadores de red disponibles en la computadora, al igual que sus direcciones IP (en caso
de tener una asignada) e ID's de hardware. En este caso podemos ver tres dispositivos:

\begin{itemize}
  \item \texttt{enp6s0}: Interfaz de ethernet, no tiene IP asignada y no ha mandado o recibido ningún paquete. Esto es consistente con el
    hecho que no está conectado ningún cable de ethernet a la computadora al momento de ejecutar el comando.
  \item \texttt{lo}: Tiene asignada la IP \texttt{127.0.0.1}, es una interfaz virtual utilizada por el sistema operativo. Este es el
    localhost.
  \item \texttt{wlp0s20f3}: Interfaz de internet inalámbria, se puede ver la IP asignada \texttt{192.168.1.17} al igual que los
    paquetes transmitidos (TX) y paquetes recibidos (RX) desde el momento de arranque. Esta es la interfaz de red activa del sistema.
\end{itemize}

Utilizando el dispositivo \texttt{wlp0s20f3} se realizó la captura con la siguiente configuración:
